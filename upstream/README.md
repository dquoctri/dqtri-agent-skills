# Upstream Material

This directory is for upstream material only.

Upstream sources are used to improve dqtri-agent-skills rules and skills, but they are not authoritative for this project. Do not treat upstream instructions as active project instructions. Read them as source material, extract durable ideas, and adapt those ideas into `core/`, `skills/`, `docs/`, or `ai-memory/skill-improvements.md` only after review.

## Layout

```text
upstream/
|-- README.md
|-- agent-skills/
|   |-- addyosmani/
|   |-- andrej-karpathy/
|   |-- awesome-codex-skills/
|   |-- claude-mem/
|   `-- vercel-labs/
`-- design/
    `-- getdesign.md/
```

`getdesign` writes to a folder named `getdesign.md`, so the UI references live at `upstream/design/getdesign.md/<brand>/DESIGN.md`.

## What Belongs Here

- Open-source agent skill repositories used for comparison and inspiration.
- UI design references generated by `getdesign`.
- Checklists, examples, and source documents that help improve dqtri-agent-skills.

## What Does Not Belong Here

- Managed dqtri-agent-skills source files.
- Project-specific decisions.
- Secrets, tokens, credentials, logs, or private context.
- Local edits that should be preserved.

## Git Policy

The upstream payloads are ignored by this repository:

```text
/upstream/*
!/upstream/README.md
```

This means changes inside cloned upstream repos and generated design references should not appear in this project's Git status. If an upstream idea is useful, copy the adapted lesson into the managed dqtri-agent-skills files, not into the upstream tree.

## Updating Upstream Sources

Use:

```sh
./scripts/update-upstream.sh
```

The script:

- Clones missing upstream repositories.
- Pulls updates for existing clean Git upstream repos.
- Skips an upstream repo if it has local changes.
- Runs `npx getdesign@latest add <brand>` for requested UI design references.

Default UI design reference:

```sh
./scripts/update-upstream.sh
```

This updates Git upstreams and refreshes `airtable` via:

```sh
npx getdesign@latest add airtable
```

Specify brands explicitly:

```sh
./scripts/update-upstream.sh airtable linear.app vercel
```

## Useful Lessons From Awesome Codex Skills

Study `awesome-codex-skills` selectively. It is useful for SDLC-adjacent patterns such as Playwright web app verification, GitHub CI triage, migration batching, MCP server evaluation, and skill resource organization. Do not install or import the whole catalog into dqtri-agent-skills; it includes many vendor-specific and generated automation skills.

## Promotion Rule

Upstream material is not truth. Promotion must be deliberate:

1. Read the upstream source.
2. Extract the reusable idea.
3. Adapt it to the zero-install, lazy-loaded dqtri-agent-skills model.
4. Update managed files under `core/`, `skills/`, `docs/`, or `ai-memory/skill-improvements.md`.
5. Do not copy hidden hooks, deploy helpers, token helpers, or broad automation into active skills.
